# Order Payments Service

Сервис работы с платежами по заказу на **FastAPI +
SQLAlchemy + PostgreSQL**.

------------------------------------------------------------------------

## Возможности сервиса

Сервис предоставляет API для работы с заказами и их оплатой:

- просмотр заказа;
- создание платежа по заказу;
- возврат платежа;
- синхронизация эквайрингового платежа с банком.

Поддерживаемые типы платежей:

- `cash`
- `acquiring`

Для эквайринговых платежей предусмотрен клиент внешнего API банка и
механизм синхронизации состояния платежа.

# Запуск через Docker

## 1. Клонировать проект

    git clone <repository_url>

## 2. Поднять контейнеры из корня проекта

    docker compose up --build -d

## 3. Накатить миграции

    docker compose exec app alembic upgrade head

## 4. Заполнить БД тестовыми заказами

    docker compose exec app python -m app.scripts.seed

## 5. Открыть Swagger

    http://localhost:8000/docs

# Основные API

## Получить заказ

    GET /api/v1/orders/{order_id}

## Создать платеж

    POST /api/v1/payments

пример:

    {
      "order_id": 1,
      "amount": "300.00",
      "type": "cash"
    }

или

    {
      "order_id": 1,
      "amount": "500.00",
      "type": "acquiring"
    }

## Возврат платежа

    POST /api/v1/payments/{payment_id}/refund

## Синхронизация банковского платежа

    POST /api/v1/payments/{payment_id}/sync

------------------------------------------------------------------------

# Бизнес‑правила

- сумма платежей не может превышать сумму заказа;
- статусы заказа:
    - `unpaid`
    - `partially_paid`
    - `paid`
- возврат возможен только для успешных платежей;
- эквайринговые платежи создаются через API банка;
- состояние платежа может измениться на стороне банка и требует
  синхронизации.
